本地安装手册

ST Character WeChat 的本地安装和配置入口。这个页面是静态离线工具，只在浏览器里读取你主动选择的文件，不上传角色卡、API 密钥、聊天记录或本地状态。

手动安装教程

你现在已经在项目根目录里。直接打开 00_START_HERE.html 就能完成本地配置，不需要进 docs/ 找工具页。运行 npm run manual 打开同一页面时，角色卡目录会由本机 helper 直接读取固定的 character-cards/。

确认 Node.js 是 22 或更高版本：node --version。

在项目根目录运行 npm install。

把你的 SillyTavern JSON / PNG 角色卡放进项目根目录的 character-cards/。终端里运行 npm run cards:list 会直接扫描这个固定目录，不需要再输入路径。

手动安装时，先进入“本地设置”确认称呼、时区和提醒；再进入“模型 / API”选择本地 Codex、本地 Claude Code 或远程 API。如果选择远程 API，在同一页填写地址、密钥并获取模型；长期记忆检索也在这里设置。最后只在“模型 / API”的“保存配置”里保存一次本地 .env。如果使用下一页的自动安装 Prompt，Codex / Claude Code 会先替你创建 .env。

运行 npm run check 做语法检查和本地安装自检，再运行 npm run login。二维码会显示在当前终端窗口里；如果终端二维码显示不完整，就打开终端里打印的链接再扫码。

运行 npm run start，在微信里发送 /char reload、/char list、/char use 1，再发送普通消息测试角色聊天。之后 Windows 用户可以直接双击项目根目录的 01_START_WECHAT.bat 启动；想从 GitHub 更新程序并启动，就双击 04_UPDATE_AND_START_WECHAT.bat。它适用于 git clone 和 GitHub ZIP 解压目录；第一次在 ZIP 目录运行会自动接到 GitHub 更新源。更新会刷新程序文件并运行 npm install，不会清理 .env、角色卡、本地记忆、报告、状态目录或 node_modules/。

如果 Windows 第一次双击 bat 时提示 Unknown Publisher，这是下载文件的安全标记，不是项目额外弹窗。点 Run 后，脚本会自动解除本目录启动文件的下载标记，后面再打开其他 bat 通常就不会继续弹。

需要开机自启动时，双击项目根目录的 02_ENABLE_STARTUP.bat。它会在 %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup 创建指向普通启动脚本的 ST Character WeChat.lnk，不会在开机时自动更新。以后想关闭自启，双击 05_DISABLE_STARTUP.bat；也可以手动删除 Startup 文件夹里的这个快捷方式。移动项目文件夹后请重新运行启用自启动的 bat。

本地设置

这里配置角色对你的称呼、你的代词、允许聊天的微信发送者、当地时间和每日提醒。模型、API 和长期记忆检索请到“模型 / API”页设置。

这里改完不用单独保存。确认本地设置后，去“模型 / API”页完成模型配置，并在那里一次写入项目根目录的 .env。

聊天身份

这些信息只写入本地 .env，用于让角色知道怎么称呼你、用什么代词，以及只回复哪个微信发送者。聊天里临时出现的新昵称不会自动写入 .env，想固定下来再回这里改。

当地时间与位置

时区下拉里显示的是常见 UTC 偏移，实际写入 .env 的是标准时区名；这样夏令时地区也不会算错。位置只写入本地 .env。

每日提醒

开启后，项目每天会在你选择的小时里随机一次提醒。如果错过这个小时，会在当天第一次回复你时自然带上天气/穿衣提醒。

模型设置

聊天模型和记忆检索模型是两组配置，但会一次保存到同一个本地 .env。切换模型或 API 不会清空角色卡、聊天记忆或微信发送者 ID；密钥只写入本地 .env，不要提交到 GitHub。

聊天模型

负责生成角色回复。选择本地 Codex 或本地 Claude Code 时，通常不用填写 API；选择远程 API 时，OpenAI、Gemini、DeepSeek 和其他兼容 OpenAI 的服务都按同一种方式填写。

当前选择的是本地运行方式，不需要填写远程聊天 API。要使用 OpenAI、Gemini、DeepSeek 或其他兼容服务，请先选择“远程 API”。

长期记忆检索

可选增强，不开也不会影响正常聊天、日期提醒和日报周报。开启后，项目会把本地记忆建成索引；每次聊天只用当前消息检索少量候选，rerank 也只排序这些候选。最终最多注入 8 条，不相关或最近刚召回过的旧记忆会被过滤或降权。

记忆检索模型

只在上面开启长期记忆语义检索时使用。它不会替代聊天模型；embedding 负责从长期本地记忆里找候选，rerank 只把这些候选重新排序。

当前未开启长期记忆语义检索，不需要填写这里。

保存配置

本地设置、聊天模型和记忆检索模型会一次写入同一个项目根目录 .env。通过 npm run manual 打开本页时会直接写入，不会生成 env.env；如果直接双击 HTML，浏览器会要求选择项目根目录。

角色卡目录

把 SillyTavern JSON / PNG 角色卡放进项目根目录的 character-cards/。这里就是本项目固定读取的角色卡文件夹。

模板能改什么

  • 内容模块：日记可改状态概览、今日小结、主要话题、关键信息、聊天摘录、补充信息、时间分布和能量趋势；周记可改状态标签、简要描述、每日事项、任务进度、睡眠和压力刻度。

  • 样式：直接写在 HTML 的 <style> 里；程序会把数据注入 __CARD_DATA_JSON__ 后截图成 PNG。
常用数据字段 需要写模板时展开查看

下面这些英文是模板里的数据字段名，写 HTML 时需要按原样使用；页面展示文字可以全部写中文。

模板库

本页会把导入的模板、当前选择和编辑草稿保存在这个浏览器里；项目真正生成长图时，读取的是模板目录里的 HTML。默认目录是 templates/cards/。

为避免覆盖内置模板，建议导出新模板文件后放进你自己建的模板目录，再在 .env 设置 ST_CHARACTER_WECHAT_REPORT_CARD_TEMPLATE_DIR。这个目录里的日记和周记文件名仍分别是 daily-diary.html、weekly-review.html。

编辑当前模板草稿

这里的修改会自动保存为浏览器草稿，不点按钮也不会因为切换分页丢掉。导出新模板文件不会覆盖项目内置模板。

输出 PNG 默认保存在本地 state 目录，不会提交 GitHub。微信命令：/dailycard、/weeklycard。

微信指令

在微信聊天里直接发送这些命令。普通文字会发给当前角色；线程、模型和报告命令通常需要先用 /char use 选好角色。

角色

/char reload重新扫描本地 character-cards/。
/char list列出当前可用角色卡。
/char use <名称|编号>切换当前微信发送者正在聊天的角色。
/char current查看当前角色和线程状态。
/char reset重置当前角色的独立对话线程。

线程与上下文

/status查看工作区、当前角色、模型和上下文用量。
/bind <绝对路径>绑定工作区；普通本地使用通常不用改。
/compact压缩当前角色线程上下文，不删除本地用户记忆。
/compact auto查看自动压缩设置。
/compact auto on/off开启或关闭自动压缩。
/compact auto <百分比>设置上下文用到多少时自动压缩。
/stop停止当前角色正在生成的回复。
/chunk <数字>调整微信消息分片合并大小；回复被切太碎时再改。

模型、授权、报告

/model查看当前角色 binding 的模型设置。
/model <id>切换当前角色使用的模型。
/yes同意当前 approval 一次。
/no拒绝当前 approval。
/always同意当前 approval；不会开启永久工作区授权。
/dailycard生成每日小结长图。
/weeklycard生成每周回顾长图。

辅助与已禁用

/help列出当前可用命令。
/star显示项目 GitHub star 提示。
/rereadcharacter-only 模式已禁用；每条消息都会重建角色 prompt。
/switchcharacter-only 模式已禁用；请用 /char use 切换角色。
/checkin /weekly /userstatus /statuscard旧命令；会提示现在应使用的替代命令。
